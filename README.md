````markdown
# Вопросы для интервью

## 1. В чём разница между `setTimeout()` и `setInterval()`? Когда использовать каждый?

**`setTimeout()`** выполняет функцию **один раз** через указанную задержку.

```javascript
setTimeout(() => {
  console.log("Выполнится через 3 секунды");
}, 3000);
```
````

**`setInterval()`** выполняет функцию **многократно через равные промежутки времени**.

```javascript
setInterval(() => {
  console.log("Выполняется каждые 3 секунды");
}, 3000);
```

### Когда использовать

| Метод           | Когда применять                                   |
| --------------- | ------------------------------------------------- |
| `setTimeout()`  | когда нужно выполнить действие **один раз позже** |
| `setInterval()` | когда действие должно **повторяться регулярно**   |

Например:

- `setTimeout` — показ уведомления через несколько секунд
- `setInterval` — секундомер или таймер

---

## 2. Что произойдёт, если не вызвать `clearInterval()`? Какие проблемы это может вызвать?

Если не вызвать `clearInterval()`, таймер **будет продолжать выполняться бесконечно**.

Это может привести к следующим проблемам:

- увеличение нагрузки на процессор
- утечки памяти
- множественные параллельные таймеры
- неправильная работа интерфейса
- ускорение таймера (если запустить несколько `setInterval`)

Пример остановки таймера:

```javascript
const timerId = setInterval(() => {
  console.log("Таймер работает");
}, 1000);

clearInterval(timerId);
```

Поэтому **любой повторяющийся таймер должен иметь возможность остановки**.

## 3. Что такое BOM? Назовите основные объекты BOM и их назначение.

**BOM (Browser Object Model)** — это набор объектов JavaScript, который позволяет взаимодействовать с **браузером**.

С помощью BOM можно:

- получать информацию о браузере
- управлять вкладками
- работать с адресной строкой
- получать параметры экрана

### Основные объекты BOM

| Объект      | Назначение                         |
| ----------- | ---------------------------------- |
| `window`    | главный объект браузера            |
| `navigator` | информация о браузере и устройстве |
| `location`  | управление адресом страницы        |
| `screen`    | информация о размере экрана        |
| `history`   | управление историей переходов      |

Пример использования:

```javascript
console.log(navigator.userAgent);
console.log(location.href);
console.log(screen.width);
```

## 4. Как узнать, с мобильного ли устройства зашёл пользователь?

Обычно проверяют строку **`navigator.userAgent`**.

Пример:

```javascript
const isMobile = /Android|iPhone|iPad|iPod/i.test(navigator.userAgent);

if (isMobile) {
  console.log("Пользователь с мобильного устройства");
} else {
  console.log("Пользователь с компьютера");
}
```

Здесь используется **регулярное выражение**, которое ищет названия мобильных устройств в `userAgent`.

## 5. Почему `alert()`, `confirm()` и `prompt()` считаются устаревшими? Какие у них недостатки?

Эти функции считаются устаревшими, потому что они **блокируют выполнение JavaScript и интерфейс страницы**.

Недостатки:

- блокируют интерфейс страницы
- нельзя стилизовать внешний вид
- ограниченный функционал
- плохой пользовательский опыт
- отличаются по стилю в разных браузерах

Пример:

```javascript
alert("Сообщение");
confirm("Вы уверены?");
prompt("Введите имя");
```

В современных приложениях обычно используют **модальные окна через HTML и CSS**.

## 6. Что делает метод `padStart()`? Зачем он нужен при работе с таймерами?

Метод **`padStart()`** добавляет символы **в начало строки**, чтобы строка имела нужную длину.

Пример:

```javascript
"5".padStart(2, "0");
// результат: "05"
```

### Почему это важно для таймеров

Без `padStart()` время будет выглядеть так:

```
5:3
```

С использованием `padStart()`:

```
05:03
```

Пример кода:

```javascript
String(seconds).padStart(2, "0");
```

Это делает формат времени **HH:MM:SS или MM:SS** более читаемым.

## 7. Как сохранить состояние таймера в `localStorage`, чтобы после перезагрузки страницы он продолжал работать?

Для этого нужно:

1. сохранять время таймера в `localStorage`
2. при загрузке страницы читать сохранённое значение

### Сохранение

```javascript
localStorage.setItem("timer", seconds);
```

### Восстановление

```javascript
const savedTime = localStorage.getItem("timer");

if (savedTime) {
  seconds = Number(savedTime);
}
```

После этого таймер можно продолжить с сохранённого значения.

## 8. Что такое `location.href`? Как перенаправить пользователя на другую страницу?

`location.href` — это **полный URL текущей страницы**.

Пример:

```javascript
console.log(location.href);
```

Он может выглядеть так:

```
https://example.com/page
```

### Перенаправление пользователя

Чтобы перейти на другую страницу:

```javascript
location.href = "https://google.com";
```

После выполнения этой команды браузер **перейдёт на новый адрес**.
